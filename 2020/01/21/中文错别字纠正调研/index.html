中文错别字识别方案调研

任务定义：NLP中的语言建模任务，在已有的句子中判断某个字的出现（困惑度）是否合理，是否符合人类的表达习惯。

相关技术方案参考

  1. 达观的解决方案：http://www.datagrand.com/blog/search-query.html
    方法：基于字之间书写过程中的“距离”，搜索可能的正确词语；基于语言模型修正。
    数据：爬虫系统爬取优质词条词典、拼音纠错词典、从检索结果是否很差推测是否有错，不需要标注语料。

  2. 百度AI提及的解决方案：https://mp.weixin.qq.com/s/r0kWgPHKthPgGqTbVc3lKw
    将错误分为3类：用词错误，音近、词近；文法/句法错误；知识性错误
    技术：基于机器翻译的方法，主要思想是把纠错看成同种语言中错误句子翻译为正确句子的过程，大规模训练语言模型后，以机器翻译任务形式进行微调训练，需要标注语料。
    纠错系统：ECNet框架，线下挖掘句对、词对齐、phrase抽取，线上检测后候选排序。Restricted-V NEC，端到端机器翻译。

  3. 腾讯内部搜索平台部提及的解决方案：https://cloud.tencent.com/developer/article/1030059
    分为召回层和决策层，召回层包含不同领域的错字检测模块，决策层基于用户、字形字音等分类，需要标注语料。

  4. 开源方案：https://github.com/shibing624/pycorrector
    基于规则：先用外部分词工具分词，若某个字的语言模型得分或某个词不在词典中，则判定错误，再基于词典、字典替换该字，使用语言模型判断最好的替换。

基于深度模型：
RNN-CRF：阿里巴巴2016中文语法纠错比赛CGED2018第一名方案
conv_seq2seq：NLPCC-2018第三名方案
BERT：基于MASK输出获得错词判断。

  1. 几种相关规则定义：
    编辑距离：由一个转成另一个所需的最少编辑操作次数
    若有先验数据，使用LSH局部敏感性哈希/KD Tree等返回候选的相似选项

语料要求

中文语料即可。该任务语料的选择与BERT相同，只需要相关领域的中文文本即可。关键在于能覆盖该领域大多数语言表达的语料库（多样性）及足够的语料数量。

学术界调研

该任务工程性较强，针对性学术研究较少。且任务语言特点较强，中文的错字常把整个字打成音似字、形似字，与英文的增删字符不同。从问题来看，属于语言模型中的子应用，与目前大规模的预训练语言模型一脉相承。因此在学术界提升上关注BERT关于词语掩码预测任务的改进即可。具体相关论文和讨论可见附录。

相关资源

  1. 开源方案4中包含同音字、形似字、错别字混淆集、人名地名等字典
  2. 字典类数据库如新华字典 https://github.com/pwxcoo/chinese-xinhua
  3. 搜狗中文字词搭配关系数据 http://www.sogou.com/labs/resource/list_lan.php

总结

  1. 基于规则与深度学习两路结合。
  2. 基于规则保证了任务早期性能基线baseline的基础，即通过开源字典、实体构建常用词库与常用搭配，匹配用户输入错字。后期再基于用户数据积累字典、词典、常错字、音似字、形似字、金融领域常用专有名词等。需要注意的是，BERT为代表的预训练模型并不能捕捉字形、字音的相似关系，所以基于形似音似等规则方法，为便捷、有效的辅助信息。
  3. 基于深度学习。BERT系列模型在预训练过程中的任务就是预测某个字/预测某个句子，对应错字识别就是根据上下文去预测这个字的内容，两者在AI上对应的思路是相同的。训练过程即我们整理语料后，微调BERT时顺便做这个任务即可。如果该任务在上两者优化后效果仍比较差，再考虑标注数据应用翻译模型等提升效果。
  4. 如果落地场景与用户搜索相关，还有许多工程上的思路。如根据搜索结果的丰富性判断是否可能输入错误；模型通过相似性计算获取与用户输入相似的表达或搭配；在用户搜索库候选召回相近的常用字段等。

附录

文本错误类型，开源的数据定义示例

• 谐音字词，如 配副眼睛-配副眼镜
• 混淆音字词，如 流浪织女-牛郎织女
• 字词顺序颠倒，如 伍迪艾伦-艾伦伍迪
• 字词补全，如 爱有天意-假如爱有天意
• 形似字错误，如 高梁-高粱
• 中文拼音全拼，如 xingfu-幸福
• 中文拼音缩写，如 sz-深圳
• 语法错误，如 想象难以-难以想象

部分相关论文

Youdao’s Winning Solution to the NLPCC-2018 Task 2 Challenge: A Neural Machine Translation Approach to Chinese Grammatical Error Correction - 网易有道
错误分为表面错误和语法错误，使用相似字音表+5-gram解决低级错误，使用字级、词级 Transformer 模型解决高级错误，将低级模型和高级模型进行组合，最终再用5-gram语言模型对纠正后句子进行困惑度分析，选择困惑度最低的句子。

Chinese Grammatical Error Correction Using Statistical and Neural Models - 阿里巴巴
多模型平行结构，使用基于规则、基于统计和神经网络三大类模型，先在类别内进行低级组合得到类别候选，再对类别候选进行高级组合。

Pre-Training with Whole Word Masking for Chinese BERT - 哈工大+科大讯飞
BERT MASK时随机MASK整个中文词语而非单字，在这里可以MASK掉容易写错的字。

Chinese Spelling Check System Based on Tri-gram Model
构造由容易被混淆的字符及词组成的混淆集 基于二元/三元文法等匹配

A Sequence to Sequence Learning for Chinese Grammatical Error Correction
根据标注语料，使用seq2seq模型生成正确序列

基于 Transformer 增强架构的中文语法纠错方法
Transformer架构的NMT机器翻译模型
