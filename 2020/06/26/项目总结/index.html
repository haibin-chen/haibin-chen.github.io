一个表格问题的探索

问题定义

无线表格预切分后存在一些误切的空格、文本，对这一类情况进行合并，方便后续表格理解任务。

模型说明

  1. 模型输入输出定义：输入为表格序列化表示，输出为每个格子是否合并。
  2. 双BERT模型。BERT的输入除了原来的embedding还有行/列embedding。（参考ACL 2020 google表格理解的模型）
  3. 第一个BERT编码字符embedding，每个格子的表示为字符embedding的平均。将每个格子的表示输入第二个BERT里，做0/1的序列标注问题。（基于BERT的LSTM/CNN/GNN并没有明显的提升）
  4. 针对多行多列合并切分/问题，使用句子对NLI问题进行辅助判断，现在比较初步的方法是，针对多行/多列问题，贪心算法计算切分点。针对N行N列问题，迭代的两行两列进行合并方向的判断。

规则放在哪里

我觉得机器学习任务，在支持以下流程的情况下，似乎可以有这样的大致套路：

  1. hard规则过滤。比如表格缺失、用户自定义业务特征等，针对字符串是不是有一些固定的正则过滤方法。（学好正则！）
  2. 机器学习模型层+soft规则过滤。在该问题中，可能有一些也许有效的soft业务特征，如是不是空格切分后存在空行？是不是文本内容占满并且写过行了？但是这些特征也不是必然和标签关系对应的。像现在基于预训练模型的特征非常强，如果只是单纯拼接到特征或FFN映射之类的，很难体现在最终效果上。组内一位算法工程师的建议是，胶囊网络对特征的引导作用很强，其他推荐系统的embedding方法好像还没找到在文档理解问题的应用方法。组内另一位同学是把深度学习预测标签+特征构造为一个机器学习问题，再用一个GBDT/XGBOOST拟合，效果也体现的比较明显。

update

最近看到了”串行化”和”并行化”的方法，上面阐述的是串行化，好处是可以快速迭代过滤大部分case，明显的可以用规则过滤，过完模型的规则也可以兜底一些长尾分布等。今天看到了”并行化”的方法，也就是有点类似于投票融合，多种规则和模型的置信度进行加权合并。

评测方式

change F1。合并前后文本内容发生合并的格子的F1。

后续提升点

  1. 其实这个问题是CV+NLP的问题，可以考虑显示地加入视觉信息。
  2. 发现finetune的BERT预训练模型，常用文本finetune后的预训练特征和表格问题并不通用，并没有带来提升。

其他点

  1. code review要做好，有时候模型的问题经常是代码的问题、然后是脏数据的问题。
  2. 经常会有这个疑问：数据究竟够了吗？现在的做法是，在确认任务设置基本合理的情况下（训练集过拟合），评估自己的模型对不同比例数据的敏感度。
  3. 构造好的数据是everything。
  4. 压测并不是做的很完整，后续要填坑。
  5. 对于tensorflow高级estimator API，可以加入Hook来获得训练过程中验证集的loss和准确率变化。
