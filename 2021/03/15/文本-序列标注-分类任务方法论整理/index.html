总论

这个博客主要整理一些文本分类、序列标注任务具体实践中的整体方法论，慢慢更新&重构。
积累从数据、模型、应用场景的思路与工具。

数据篇

训练数据要多少？

众所周知，数据越多越好。那么，如何衡量数据量增加的边际效应递减呢？如何考量增加什么数据呢？

  1. 做增量学习实验。总体而言，深度学习模型的边际效应往往类似于对数回归模型，f(x)=alogb+c。当然也可以使用专门描述边际效应的一些模型如渐进回归模型。选取若干个数据点进行实验，在感官上和量化上控制数据要求。
  2. 任务特定的场景下，根据效果提升的瓶颈来决定是否增加训练数据、还是增加词典/pattern匹配。对于实体识别来说，特定实体的规律性（人名大小写，地点名后接路字）、实体是否在训练集收集到、上下文表达形式是否有多样性，都是决定分类结果的关键因素。而对于实体，是可以利用词典如Gazetteers等先验信息。因此，当上下文丰富性达到一定程度时，数据量增加带来的提升往往是“记住”某些新的实体名字，而这部分是可以通过低成本获得的词典来做增强的。[1]

[1] A Rigorous Study on Named Entity Recognition: Can Fine-tuning Pretrained Model Lead to the Promised Land? (2020 AAAI)

怎么做数据增强？

  1. 远程监督。如对于极少量数据的目标域，如只有200个句子标注的实体数据集。收集实体维基百科、相关主题语料远程监督等，在这个目标域上进行预训练模型的微调，还是非常有效的方案。[1]
  2. 半监督。利用已有模型在未标注数据上标伪标签，迭代训练。也可以利用弱监督相关信息。[2]
  3. 替换、翻转等方法。如同义词替换(词典/语言模型)、随机插入、随机交换和随机删除等等。文本分类任务简单、成熟一点。代表工作有UDA[3]、实体识别相关[6]等。
  4. 回译。
  5. 基于Mixup的方法，利用插值等思路生成新样本。[4]
  6. 生成式数据增强。常见思路是先微调预训练模型做原句子copy任务，模型生成句子保留大致含义但有多样性。[5]
    个人经验，靠上的方法相对来说更通用有效一些。
    其他相关问题：带噪声的学习
    [1] CrossNER: Evaluating Cross-Domain Named Entity Recognition (2021 AAAI)
    [2] Fine-Tuning Pre-trained Language Model with Weak Supervision: A Contrastive-Regularized Self-Training Approach (2021 NAACL)
    [3] Unsupervised Data Augmentation for Consistency Training (2020 NIPS)
    [4] Local Additivity Based Data Augmentation for Semi-supervised NER (2020 EMNLP)
    [5] Data Augmentation using Pre-trained Transformer Models
    [6] An analysis ofsimple data augmentation for named entity recognition (2020 COLING)

拿到数据后可以做哪些初步分析？要怎么清洗？

  1. 在标注过程和清洗过程中，需要确定一个比较统一的标准，避免混淆。如标注人名“张老师”，“老师”两个字要不要认定为实体？这个是需要具体分析确定的。
  2. 样本分布是否均衡？有些类是否可以合并为异常（其他）类？

如何找到脏数据？怎么做带噪声学习？

  1. 在NLP中往往是先拟合特征再拟合噪声[1]，可以用early stopping的模型去预测数据集做筛选
  2. 对于CRF来说，可以是partial化即可能的路径都视为正确[2]，这个思路还可以用在远程监督等
  3. 通用的噪声学习/半监督学习的一些技巧和原则，如熵最小化、置信度估计[3]等。
    [1] BERT memorisation and pitfalls in low-resource scenarios
    [2] Better Modeling of Incomplete Annotations for Named Entity Recognition (2019 NAACL)
    [3] Noisy-Labeled NER with Confidence Estimation (2021 NAACL)

数据清洗

  1. 可以将专有名词或者某类无关专业名词改成符号表示或模板表示。
  2. vocab取前N个高频词或过滤次数小于3的词等。
  3. 文本纠错：常见的就是维护一个规则表转换错别字全半角等，使得数据满足特定格式。
  4. 若数据很脏如互联网微博，可以考虑结合char和word模型，另外，可以用特殊超参小窗口2-3的fasttext去训练一份词向量，也更好地学习了n-gram subword信息。

标签体系构建

如何构建标签是实际应用里非常重要的部分，以下写出一些建议：

  1. 长尾标签可设置为“其他”，或者看情况删除。
  2. 是否存在易混淆标签？这个可以从bad case分析和梯度分析中看出来。
  3. 多标签问题，标签间是否存在层次化关系，是否有分多步的关系？
  4. 极多标签问题，如电商上万标签的商品分类，思路有类似搜广推的地方，都是粗排+细排的结构。

什么样叫低资源小数据？

个人主观经验很强的结论：

  1. 对于文本分类而言，如果是n<6-7类且问题不算特别复杂，500条以下数据算是比较少的，可以初步验证下限，此时数据增强非常有效，包括用语言模型做数据增强。模型上总体而言预训练模型还是比较鲁棒。
  2. 对于序列标注而言，1000条句子，n<4-5类以下算是比较少的，同样可以初步验证下限。但序列标注比文本分类的标签类型稍微复杂一些，用语言模型做数据增强效果会比较差，模型上可以考虑小样本的网络。

数据挖掘

  1. 难例挖掘方法
  2. 常用正则表达式的使用
  3. 常用句法分析工具，结合主谓结构，动宾结构，补语结构等等，找出我们所需的信息单元。

模型篇

规则模板

常见的算法策略，可以结合hard规则(要么是、要么不是)和soft规则(可以考虑特征化，或按一定比例加权)来给模型做兜底，提升前期召回率等等。

机器学习模型

如果样本量大、需要一定可解释性，可以使用LR。特征维数多，就可以借助基于核函数的SVM。

深度学习模型

  1. 如果使用BERT类模型，在领域数据上增量预训练。

NER少样本快速启动指南

AutoNER、弱监督方法。
高质量短语的挖掘可以用AutoPhrase。
自训练self-training

文本分类少样本快速启动指南

  1. 可以考虑借助预训练语言模型做词库积累[1]
  2. 有显著特点、且数据集只有几十几百条这种可怜的情况，可以用匹配的思路去尝试。如文章[2]，大概思路是类别名->做为种子获得更多词(看冲突否)->attention获得类别表示->聚类文档和类别并对齐，聚类的使用非常自然和合理，弱监督的思路。
    [1] Text Classification Using Label Names Only: A Language Model Self-Training Approach
    [2] X-Class: Text Classification with Extremely Weak Supervision 2020 EMNLP

    针对中文NER的词典增强

    Lattice-LSTM为主线，通过图网络等方式将词典信息在模型中显式建模。

CRF解码速度慢怎么办？

  1. LAN：基于标签语义attention进行refine，效果提升不确定。
  2. Uncertainty-Aware Label Refinement for Sequence Labeling：通过Bayesian神经网络估计模型不确定性，再做refine。

如何利用超大(预训练)模型?

如果资源够丰富，先用NAS神经网络架构搜索搜个大网络，在小网络上（也可NAS搜）把大网络当成teacher监督，引入学习teacher soft target输出的辅助任务。这里的知识蒸馏用最经典的soft target熵值比较，和特征表示MSE损失的方法就可以了。后面就是(量化)部署了。

新词发现

BERT根据互信息来做无监督分割/远程监督召回样本，将候选序列与实体词典的交集作为训练正例样本，通过多轮迭代筛选高置信度标签。
注意事项：

  1. 记得提高阈值保证精读而牺牲召回。
  2. 结合统计指标和规则要求筛选新实体，如最低频率、互信息等。
