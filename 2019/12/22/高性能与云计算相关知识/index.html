高性能与云计算

PPT核心内容

并行计算机体系结构

  1. 时延：软件、通道、选路、竞争；对剖宽度:将网络分成两个相等部分所必须移去的最少边数；对剖带宽:每秒钟内， 在最小的对剖平面上通过所有连线的最大信息位(或字节)数。等于对剖宽度与链路带宽之积
  2. 静态互连网络：点到点的链接不变 如线性阵列 与动态互连网：交换开关构成 如总线
    对剖带宽表、直径表
  3. 对剖带宽愈大，表示网络的带宽就愈大;网络直径愈大，表示通信的时间延迟就愈大。网络的造价随节点度和链路数增大而上升。固定网络规模，对剖宽度越大，或网络直径越小，则互联网 络质量越高
  4. MIMD结构：共享存储MIMD / Multiprocessor；分布式存储
  5. 多处理机：UMA模型是均匀存储访问模型的简称，物理存储器均匀共享；多计算机：NUMA(NCC-NUMA)模型是非均匀存储访 问模型的简称，物理上分布在所有处理器；COMA模型是高速缓存存储访问模型，全部高速缓存 组成了全局地址空间；CC-NUMA模型是高速缓存一致性非均匀存储访问模型；NORMA模型是非远程存储访问

并行计算机系统及性能评测

  1. 对称多处理机SMP：单地址空间、高速缓存及一致性、对称性
    CC: Cache Coherent-NUMA: Non-Uniform Memory Access
  2. 大规模并行处理机MPP：多个微处理器、分布存储器
  3. 集群(机群)是一组独立的计算机(节点)的集合体
  4. 性能评测基本概念：机器规模n 工作负载W 加速 Sn=T1b顺序/Tn并行 效率 En = S n/n
    性能speedup = workp/timep / work1/time1
  5. Amdahl 定律:计算负载不变 S = ws+wp / ws+wp/p = f+1-f / f+(1-f)/p = p / 1+f(p-1)
    Gustafson定律:时间固定 S = ws+pwp / ws+pWp/p = ws+pwp / ws+wp or S = f+p(1-f)
    Sun & Ni定律:存储受限 S = fw+(1-f)G(p)w / fw+(1-f)G(p)W/p
    G(p)=1时就是Amdahl加速定律, G(p)=p 变为 f + p(1-f )，就是Gustafson加速定律,G(p)>p时，相应于计算机负载比存储要求增加 得快，此时 Sun Ni 加速均比 Amdahl 加速和 Gustafson 加速为高

并行算法设计(1)

  1. PRAM (Parallel Random Access Machine):有一个集中的共享存储器和一个指令控制器， 通过共享存储(SM)的Read/Write交换数据 ，隐式同步计算 ,单一地址空间,不适合MIMD并行机，忽略了SM的 竞争、通讯延迟等因素
  2. 存取模式：CRCW:并发读并发写，冲突解决模式: 相同/公共并发读写(Common CRCW):仅允许
    写入相同数据；优先并发读写(Priority CRCW):仅允许优先级 最高的处理器写入；任意并发读写(Arbitrary CRCW):允许任意处 理器自由写入
  3. 块同步模型(Bulk Synchronous Parallel )BSP：异步MIMD-DM(分布式存储)模型，支持 消息传递系统，块内异步并行，块间显式同步。3个组成部分：节点、通信网络、同步障。强调了计算任务和通信任务的分开，即桥模型；显示同步效率低。
  4. LogP模型分布存储的、点 到点通讯的多处理机模型，异步，隐藏并行机网络结构，难以算法描述设计

并行算法设计(2)

  1. 平衡树：树叶结点为输入，中间结点为处理结点，由叶向根或由根向叶逐层并行处理
    前缀和：数组B记录由叶到根正向遍历树中各结点的信息-求和，数组C记录由根到叶反向遍历树中各结点的信息-播送前缀和
  2. 倍增技术：当递归调用时，所要处理数据之间的距离逐步加倍，经过k步后即可完成距离为2k的所有数据的计算。
  3. 分治策略
  4. 划分原理：划分成p个独立的规模几乎相等的子问题
  5. 流水线技术
  6. 并行算法的一般设计过程：划分（域分解、功能分解）、通信、组合、映射

    并行程序设计基础

  7. 矩阵乘法计算方法
  8. 并行编程风范:
    (1) 相并行：一组超级步，步内各自计算，步间通信、同步
    (2) 分治并行
    (3) 流水线并行 一组进程
    (4) 主-从并行 主进程:串行、协调任务 子进程:计算子任务
    (5) 工作池并行
  9. SIMD:所有进程在同一时间执行相同的指令;MIMD:各个进程在同一时间可以执行不同的 指令;(SPMD(Single Program Multiple Data ):各个进 程是同构的，多个进程对不同的数据执行相同的代码;MPMD(Multiple Program Multiple Data ):各个 进程是异构的， 多个进程执行不同的代码)
  10. 并行程序设计模型: 隐式并行、显示并行、数据并行、共享存储

共享存储编程

  1. 共享存储编程模型：所有线程可以存取相同的、 全局共享的存储空间，数据也可私有
  2. 线程与进程：多个进程的内部数据和状态都是完全独立的， 而多线程是共享一块内存空间和一组系统资源, 有可能互相影响；线程本身的数据通常只有寄存器数据，以及一 个程序执行时使用的堆栈，所以线程的切换比 进程切换的负担要小
  3. OpenMP应用编程接口API是在共享存储体系结构上的一个编程模型，三个API分量：编译制导、运行库例程、环境变量
  4. Fork:在并行结构的开头，主线程创 建了一队的并行线程，并行域中的代码在不同 的线程对中并行执行
    Join :在并行结构的末尾，所有线程同步或终止，只保 留主线程
  5. 可编程控制并行、支持动态编程、支持嵌套并行化
  6. 编译制导 (Directive)语句格式 #pragma omp 制导指令前缀都需要、制导指令、子句。如：#pragma omp parallel default(shared) private(beta, pi)
    #pragma omp for 数据并行；#pragma omp sections 功能并行..

2. MapReduce实现机制  
划分输入数据为分片 -> 创建master和worker线程 -> map分配到worker -> 处理后输出并通知master -> master传递存储位置到reduce -> reduce排序并追加输出 -> master唤醒进程  
负载均衡: 拖尾(Stragglers)是指那些在做最后的map或reduce时用 时超长的任务: 任务调度备份机制并重新调度运行；周期性地ping worker  

### 云计算平台  
1. 云计算的三种服务模式: 软件即服务(SaaS) 平台即服务(PaaS) 基础架构即服务(IaaS)
2. 云计算基础：虚拟化，将服务器物理资源抽象成逻辑资源。虚拟化的类型：拆分、整合、迁移  
VMM:全虚拟化（VMware）、半虚拟化（修改操作系统内核 性能高）、硬件虚拟化  
3. GFS将容错的任务交给文件系统完成，利用软 件的方法解决系统可靠性问题；用户访问Master节点，获取交互的 Chunk Server信息，实现控制流和数据流的分离，且IO高度并行，降低了master的负载
